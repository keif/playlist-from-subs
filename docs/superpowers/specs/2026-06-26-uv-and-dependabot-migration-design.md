# uv Migration & Dependabot Setup

**Date:** 2026-06-26
**Status:** Approved

## Summary

Migrate `yt-sub-playlist` from `pip + requirements.txt + venv` to [`uv`](https://docs.astral.sh/uv/) for Python dependency management, and add a `.github/dependabot.yml` so Dependabot keeps the Python stack patched. Version bumps (driven today by `standard-version` via `package.json`) get a small custom updater so `pyproject.toml` stays in sync.

## Goals

- Deterministic installs via a committed `uv.lock`.
- Single command for contributors: `uv sync` (or `uv sync --extra dashboard` for the web UI).
- Dependabot opens grouped, conventional-commit-prefixed PRs for Python dependency updates.
- `npm run release` continues to be the single entry point for version bumps and updates both `package.json` and `pyproject.toml`.

## Non-goals

- Adding an automated Python test suite. The repo has none today; introducing one is out of scope. Verification for this migration is manual smoke testing (CLI launches, dashboard backend starts, dry-run release bumps both manifests).
- Monitoring npm / pnpm / GitHub Actions ecosystems via Dependabot. Python only for this round.
- Restructuring the package layout, build artifacts, or runtime configuration.

## Current state

- `requirements.txt` lists nine pinned packages — five core CLI, three Flask dashboard deps separated by a comment, plus `python-dotenv`.
- `Werkzeug==3.0.13.11.8` in `requirements.txt` is not a real PyPI version (typo). This needs to be fixed during the migration.
- `.python-version` pins `3.11.8`. uv reads this natively.
- `venv/` is a local virtualenv checked into the repo's working tree (gitignored).
- `package.json` + `pnpm-lock.yaml` exist purely to power `standard-version` for changelog/version bumps. No application JS.
- No `.github/` directory exists. No CI today.

## Design

### `pyproject.toml`

PEP 621 metadata with a `dashboard` optional extra for the Flask backend.

```toml
[project]
name = "yt-sub-playlist"
version = "4.1.0"
description = "Automatically sync YouTube subscription uploads into a custom playlist"
readme = "README.md"
license = { file = "LICENSE" }
requires-python = ">=3.11"
dependencies = [
  "google-api-python-client==2.116.0",
  "google-auth==2.27.0",
  "google-auth-oauthlib==1.2.0",
  "google-auth-httplib2==0.2.0",
  "python-dotenv==1.0.1",
]

[project.optional-dependencies]
dashboard = [
  "Flask==3.0.0",
  "Flask-CORS==4.0.0",
  "Werkzeug==3.0.6",   # corrected from invalid 3.0.13.11.8 in requirements.txt
]

[project.scripts]
yt-sub-playlist = "yt_sub_playlist.__main__:main"

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"
```

Notes:
- `Werkzeug` pin is corrected to a real version compatible with Flask 3.0. Implementation step should verify the exact compatible pin against Flask 3.0.0's constraints.
- `[project.scripts]` is conditional on `yt_sub_playlist/__main__.py` exposing a callable `main()`. If not, expose one or drop the scripts table (users keep using `python -m yt_sub_playlist`). Confirm during implementation.
- `hatchling` is chosen as a minimal, modern PEP 517 backend with no project-specific config required.

### `standard-version` updater for pyproject.toml

`standard-version` ships no built-in TOML updater. A small custom updater under `scripts/bump-pyproject.js` exposes `readVersion(contents)` and `writeVersion(contents, version)`. Regex-based, anchored to the first `version = "x.y.z"` line after the `[project]` table header. Approximate shape:

```js
// scripts/bump-pyproject.js
const VERSION_RE = /(\[project\][\s\S]*?\n\s*version\s*=\s*")([^"]+)(")/;

module.exports = {
  readVersion(contents) {
    const m = contents.match(VERSION_RE);
    if (!m) throw new Error("version not found under [project] in pyproject.toml");
    return m[2];
  },
  writeVersion(contents, version) {
    return contents.replace(VERSION_RE, `$1${version}$3`);
  },
};
```

Wired into `package.json`:

```json
"standard-version": {
  "bumpFiles": [
    { "filename": "package.json", "type": "json" },
    { "filename": "pyproject.toml", "updater": "scripts/bump-pyproject.js" }
  ]
}
```

Trade-off accepted: regex is simpler than a full TOML parser and avoids round-trip formatting damage; the risk of a second `version = ` line under another table is mitigated by anchoring the match to `[project]`.

### `.github/dependabot.yml`

```yaml
version: 2
updates:
  - package-ecosystem: "uv"
    directory: "/"
    schedule:
      interval: "weekly"
      day: "monday"
    open-pull-requests-limit: 5
    groups:
      google-auth:
        patterns:
          - "google-auth*"
          - "google-api-python-client"
      flask:
        patterns:
          - "Flask*"
          - "Werkzeug"
    commit-message:
      prefix: "chore(deps)"
      include: "scope"
    labels:
      - "dependencies"
```

Choices:
- **Ecosystem `uv`** — native Dependabot support updates `uv.lock` directly, avoiding lockfile drift.
- **Weekly Monday** — predictable cadence, not noisy.
- **Grouping** — bundles `google-auth*` + `google-api-python-client` together, and `Flask*` + `Werkzeug` together. Reduces PR churn and avoids partial-upgrade test failures.
- **`chore(deps)` prefix** — matches Conventional Commits required by `standard-version` so the changelog stays clean.
- **Limit 5** — sane default; revisit if it gets noisy.

## README install section (after migration)

```bash
git clone https://github.com/yourusername/yt-sub-playlist.git
cd yt-sub-playlist
uv sync                       # CLI only
uv sync --extra dashboard     # CLI + Flask dashboard
```

Run commands:
```bash
uv run python -m yt_sub_playlist          # or `yt-sub-playlist` if the script entry lands
uv run python -m yt_sub_playlist.auth.oauth
```

## Verification (manual smoke)

1. `uv sync` — confirm `.venv/` is created and core CLI deps resolve.
2. `uv run python -m yt_sub_playlist --help` — confirm CLI entry still works.
3. `uv sync --extra dashboard` — confirm Flask deps install cleanly.
4. Start the dashboard backend (per `dashboard/backend/`) — confirm it boots.
5. `npm run release:dry-run` — confirm both `package.json` and `pyproject.toml` show projected version bumps.
6. Push branch with `.github/dependabot.yml`; verify GitHub's Insights → Dependency graph → Dependabot tab accepts the config without errors.

## Risks & mitigations

- **Werkzeug pin correction** could surface a runtime incompatibility with Flask 3.0.0. Mitigation: verify against Flask 3.0.0's install_requires during implementation; smoke-test the dashboard.
- **`[project.scripts]` entry** may not match a real `main()` callable in `__main__.py`. Mitigation: inspect during implementation; remove the scripts table if no callable exists.
- **Dependabot's `uv` ecosystem is relatively new.** Mitigation: if Dependabot fails to parse the config, fall back to `package-ecosystem: "pip"` against `pyproject.toml` and accept that `uv.lock` will need a local `uv sync` after each merge.
- **`standard-version` regex updater could mis-match** if pyproject grows additional `version = ` lines under other tables. Mitigation: the regex anchors to `[project]`; covered by smoke test #5.

## Out of scope

- Adding a pytest harness or CI workflow. Worth its own spec.
- npm/pnpm or GitHub Actions Dependabot ecosystems.
- Migrating the Node tooling itself off pnpm.
- Restructuring `yt_sub_playlist/` package layout.
